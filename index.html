Ubuntu Study

文件管理方式和windows差异较大，类似树的形式，没有盘符
ls：list，列出当前目录下的所有文件，配合参数使用，ls -a
ln:软链接
sudo:超级用户权限
cd
mkdir
rmdir，均用于目录
rm -rf：移除目录下所有文件
sudo rm -rf /*（bushi
group：用户组，没想到什么用处
用户：
useradd
passwd
userdel
tar文件：
tar文件是linux中很常见的一种文件，target表示归档，未压缩仅打包
.tar文件仅打包未压缩，可以使用以下命令行解包
1.tar -xvf name.tar
2.解压gz文件，.gz是单个文件压缩（没有包）
gzip -d name.gz
3.解压.tar.gz（或者.tgz)
最常见，tar -zxvf name.tar.gz
(指定目录就添加路径参数：tar -C /file -zxvf name.tar.gz)
4.解压zip
unzip name.zip
unzip name.zip -d/path
(需提前安装unzip工具。sudo apt-get install unzip)
5.5. 解压 .rar 文件

.rar 格式需要额外安装工具：

sudo apt-get install rar unrar
解压命令如下：

unrar x FileName.rar

改变不同用户关于文件权限：
chmod -a(或者u）+（或者-）权限 文件名
改变文件归属
chown 用户名 某文件（仅仅在sudo或者root下可以）
脚本：
linux主要使用shell与python脚本
shell解释器：/bin/sh，python解释器：/bin/python3
脚本具有可执行性（不知道为什么我创建的文件没有，使用chmod a+x）
shell脚本：linux自带，相当于Windows的DOS批处理脚本
编辑一个文本文件，保存为*.sh,
#!/bin/sh
echo “ “
添加可执行权限
chmod +x .sh
./.sh执行程序
（shell语言教程需要自学）
同理有python，第一行都是#！/bin/,#!加上解释器路径，必须要有x权限才能执行，执行程序时必须加上程序的路径才行，可以直接输路径执行或者调用解释器
python脚本创建：文本文件，写入代码，添加可执行权限，执行
简单的shell知识
定义一个变量，直接给名字赋值，等号两边不要多加空格，使用一个变量 ${name},如果有定界符，则简写为$name,例如echo $name/build（主要是为了区分）
接下来过渡到环境变量
根据个人理解，环境变量类似于编程语言中的全局变量，
export name=,创建环境变量，调用方式与变量类似，定义后可直接在当前命令行使用，但其余命令行和重启系统会消失。
不消失的方法：
用户环境变量：定义在/.profile中
tip：在linux中，所有以.开头的为隐藏文件，所以默认情况下打不开.profile文件（一个sh脚本）
方法，首先用文本编辑器打开/.profile， 个地图~/.profile,然后，添加变量，export Name=，注销，重新登录后生效。仅针对当前用户有效，对其余用户不可见。
tips：有的linux系统是.bash-profile
系统环境变量：都可见了
存放在/etc/profile下，使用root权限，gedit /etc/profile,一般不建议直接修改，而是在profile,d下创建一个脚本，递归调用
gedit/etc/profile.d/***.sh
创建环境变量，export
重新启动
PATH环境变量（碰到最多的）
echo #PATH会跳出大量目录，搜索可执行性脚本的路径，比如说/usr/sbin，/usr/bin，/usr/local/bin，诸如此类，这些目录是没有主文件夹的，所以我们自己写的脚本往往不会直接运行，所以必须在前面添加路径（.表示当前路径）
接下来演示如何修改PATH环境变量例如系统变量，/etc/profile.d/**.sh
编辑hello.sh，设定PATH环境变量
export PATH=$PATH:路径，重启后就可以直接使用名字了（先把原来的值取出来，冒号是分隔符，再附加后面的路径

sudo ifconfig xxx up 。sudo ifconfig xxx down。开启禁用某端口
